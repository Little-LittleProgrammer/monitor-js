# 打包原理

## 概述

Quantum Monitor 采用基于 Rollup 的现代化打包方案，支持多格式输出、TypeScript 编译、代码优化、体积监控等功能，实现了高效的 monorepo 构建流程。

## 打包架构

### 1. 构建工具链

```
┌─────────────────────────────────────────────────────────────┐
│                     构建工具链                               │
├─────────────────┬───────────────┬───────────────┬───────────┤
│     Rollup      │  TypeScript   │   Terser      │  Plugins  │
│   (核心打包)     │  (类型编译)    │  (代码压缩)    │  (功能扩展) │
└─────────────────┴───────────────┴───────────────┴───────────┘
```

#### 1.1 核心工具

- **Rollup**：专为库文件设计的打包工具
- **TypeScript**：类型安全的 JavaScript 超集
- **Terser**：JavaScript 代码压缩工具
- **API Extractor**：TypeScript 声明文件合并工具

#### 1.2 插件体系

```typescript
// rollup.config.js 插件配置
plugins: [
  resolve(),           // 模块解析
  size(),             // 体积监控
  visualizer(),       // 打包分析
  commonjs(),         // CommonJS 支持
  json(),             // JSON 文件支持
  cleanup(),          // 代码清理
  typescript(),       // TypeScript 编译
  terser()           // 代码压缩 (生产版本)
]
```

## 构建脚本

### 1. 主构建脚本

#### 1.1 并行构建实现

```javascript
// scripts/build.js
async function runParallel(maxConcurrency, sources, iteratorFn) {
  const ret = [];
  for (const item of sources) {
    const p = Promise.resolve().then(() => iteratorFn(item));
    ret.push(p);
  }
  return Promise.all(ret);
}

function buildAll(targets) {
  runParallel(10, targets, rollupBuild);
}
```

#### 1.2 单包构建流程

```javascript
async function rollupBuild(target) {
  const pkgDir = path.resolve(`packages/${target}`);
  const pkg = require(`${pkgDir}/package.json`);
  
  // 1. 检查是否为私有包
  if (pkg.private) return;
  
  // 2. 构造 Rollup 命令参数
  const args = [
    '-c',
    '--environment',
    [`TARGET:${target}`, `TYPES:${buildTypes}`, `LOCALDIR:${LOCALDIR}`]
      .filter(Boolean)
      .join(',')
  ];
  
  // 3. 执行 Rollup 构建
  await binRun('rollup', args);
  
  // 4. 复制文件到示例目录
  if (['browser', 'vue'].includes(target)) {
    const _file = path.resolve(`packages/${target}/dist/${target}.min.js`);
    const _dest = path.resolve(`example/js/${target}.min.js`);
    fs.copyFileSync(_file, _dest);
  }
  
  // 5. 生成类型声明文件
  if (buildTypes && pkg.types) {
    const { Extractor, ExtractorConfig } = require('@microsoft/api-extractor');
    const extractorConfigPath = path.resolve(pkgDir, `api-extractor.json`);
    const extractorConfig = ExtractorConfig.loadFileAndPrepare(extractorConfigPath);
    const extractorResult = Extractor.invoke(extractorConfig, {
      localBuild: true,
      showVerboseMessages: false
    });
  }
}
```

### 2. 版本管理脚本

#### 2.1 版本同步机制

```javascript
// scripts/version.js
async function modify_version(pkgName, version) {
  const pkgRoot = getPkgRoot(pkgName);
  const pkgPath = path.resolve(pkgRoot, 'package.json');
  const pkg = require(pkgPath);
  
  // 1. 更新包版本
  if (pkg.name.startsWith(SDK_PREFIX)) {
    pkg.version = version;
  }
  
  // 2. 更新内部依赖版本
  const dependencies = pkg.dependencies || {};
  Object.entries(dependencies).forEach(([dependent, dependentVersion]) => {
    const isExist = beModifiedPackages.some(
      (pkg) => `${SDK_PREFIX}/${pkg}` === dependent
    );
    if (isExist) {
      dependencies[dependent] = `^${version}`;
    }
  });
  
  // 3. 写入文件并格式化
  fs.writeFileSync(pkgPath, JSON.stringify(pkg));
  await binRun('prettier', ['--write', pkgPath]);
}
```

#### 2.2 体积检测

```javascript
// scripts/utils.js
exports.sizeCheck = async(beReleasedPackages) => {
  let _flag = true;
  try {
    if (beReleasedPackages.includes('wx-mini') || 
        beReleasedPackages.includes('browser')) {
      await execa('yarn', ['size'], { stdio: 'inherit'});
      console.log(chalk.green(`Size check passed`));
    }
    _flag = true;
  } catch (error) {
    console.error(chalk.red('package size exceeds limit'));
    _flag = false;
  }
  return _flag;
};
```


## 最佳实践

### 1. 包开发规范

- 每个包必须有独立的 `package.json`
- 私有包设置 `"private": true`
- 明确声明外部依赖

### 2. 构建策略

- 优先使用 ESM 格式
- 生产环境必须压缩代码
- 定期进行体积检测

### 3. 版本管理

- 统一版本号管理
- 内部依赖版本同步
- 构建前体积检测

### 4. 性能优化

- 合理配置外部依赖
- 启用 tree shaking
- 使用并行构建

## 总结

Quantum Monitor 的打包系统通过以下特性实现了高效的构建流程：

1. **多格式支持**：满足不同环境的使用需求
2. **类型安全**：完整的 TypeScript 类型声明
3. **模块化设计**：支持按需加载和 tree shaking
4. **自动化流程**：从构建到发布的全自动化
5. **性能优化**：并行构建和增量更新
6. **质量保证**：体积检测和代码分析

这套打包方案为 monorepo 架构的前端监控 SDK 提供了稳定、高效的构建能力。
